# MOF Synthesis Advisor — v0.1 README

Status: **shipped 2025**.  Phase 3.5 closeout.

Single-page Streamlit app that predicts solvent cluster, reaction
temperature, and synthesis method for a user-specified MOF recipe
(metal source + linker(s) + optional modulator).

---

## What it does

| Head | Output | Eval (scaffold split) |
|---|---|---|
| **Solvent classifier** | top-3 solvent clusters w/ calibrated confidence | top-1 = 0.450 · cov@90% = **0.104** |
| **Temperature** (two-stage) | predicted reaction T in °C | within ±10 °C = **0.306** · MAE = 31.7 °C |
| **Method classifier** | top-3 synthesis methods w/ calibrated confidence | top-1 = **0.749** · cov@90% = **0.717** |
| **k-NN provenance** | 3 nearest training recipes (Morgan-FP cosine) | — |
| **OOD-metal gate** | warns when metal element ∉ training distribution | — |

Validated on a 5-MOF canonical regression harness
(MOF-5, HKUST-1, UiO-66, ZIF-8, MIL-101): **4/5 pass**, 1 documented
xfail (MIL-101 high-T Cr-solvothermal is sparse in training).
See [other/canonical_tests_report.md](../Claude_auto_MOF_script/other/canonical_tests_report.md).

## Quickstart

```bash
cd /Users/mac/Documents/MOF

# (one-time) build the model checkpoint (~10 s, 0.9 MB output)
python3 Claude_auto_MOF_script/scripts/build_v01_checkpoint.py

# launch the advisor (cold start ~6 s with joblib bundle)
python3 -m streamlit run Claude_auto_MOF_script/scripts/advisor_v01_app.py
```

Open the printed `http://localhost:8501` URL. Enter a metal source,
comma-separated linker(s), optional modulator, click **Predict**.

To re-validate after any data or code change:

```bash
python3 Claude_auto_MOF_script/scripts/canonical_tests.py   # exit 0 = clean
```

## Model details

- **Backbone:** XGBoost (depth 4, n_estimators 150, learning_rate 0.1,
  `tree_method="hist"`)
- **Calibration:** Platt sigmoid via `CalibratedClassifierCV` over a
  `FrozenEstimator` (sklearn ≥1.6)
- **Feature pipeline:** 2 203 dims = 138 base recipe features (top-50
  linkers, top-30 modulators, metal symbol, amount stats) + 17
  non-Morgan chem features + 2 048 averaged Morgan-2 fingerprints
- **Splits:** all reported metrics are scaffold-split train→test (no
  leakage); the shipped model is the same one evaluated, trained on
  scaffold-train + scaffold-val calibration set.
- **Training cost:** ~10 s on M1 Max for all 3 heads + KNN index.

## KNOWN LIMITATIONS

> The same caveats are surfaced in the chemist-facing user guide
> (`v0.1_chemist_guide.md`). They are not bugs — they are the honest
> envelope of v0.1.

1. **Coverage on novel scaffolds is low.** Solvent cov@90% on the
   scaffold-test split is **~10 %**. Top-3 (not top-1) is the
   primary use mode.
2. **The time-h head is a stub.** It reports the training-data
   median `time_h` for the predicted solvent cluster. Treat it as
   "literature norm", not a model prediction. A dedicated head is
   deferred to v0.1.1.
3. **Method head shipped but coverage-gated.** Top-1 = 0.749 and
   cov@90% = 0.717 on the scaffold split. Outside its high-confidence
   regime the model still defaults toward `solvothermal` because the
   training prior is 53 % solvothermal.
4. **OOD warning fires loudly on unseen metals.** When the parsed
   metal element is not in the training metal distribution
   (`build_chem_features.METAL_PT`), the UI surfaces a yellow banner
   and predictions should be treated as guesses.
5. **Training data is literature-only.** Public MOF papers report
   syntheses that worked. We cannot predict failure modes, yields,
   or phase purity — only what literature *did* report.
6. **Modulators outside the training top-30 are silently dropped.**
   E.g. HF in MIL-101 isn't represented; this is why MIL-101 is
   xfail in the canonical tests.
7. **No molecular-property prediction.** The advisor recommends
   *conditions*, not whether a target MOF will form.

## Methodology contributions (Phase 3 highlights)

- **Pre-training resolution audit** before training any head
  (Phase 3.4.2 — flagged the linker-SMILES gap that would have
  silently capped solvent accuracy)
- **MAE/within-tolerance dissociation** as a temperature-head
  diagnostic (Phase 3.4.3): single-stage MAE looked fine but the
  ±10 °C rate was 0.232 because the regressor compressed to a global
  mean. Two-stage RT-vs-heated splitter recovered this to 0.306.
- **Calibrated-abstention + provenance design** — every prediction
  ships with a confidence number and the 3 nearest training rows so
  the chemist can audit "why".
- **NULL recovery — published negative result.** Documented in
  `decision_log.md`: targeted NULL-imputation for missing solvents
  *underperformed* the simpler "drop NULL" baseline. Reproduced
  across three configurations. Comment-level finding for a methods
  paper after v0.1 stabilizes.

## Honest "what we cannot do"

- We **cannot** predict synthesis failure or yield.
- We **cannot** predict crystal phase, defect density, or surface area.
- We **cannot** extrapolate to metal elements absent from the
  training set (the OOD gate fires; predictions are unreliable).
- We **cannot** model modulators outside our top-30 vocabulary.
- The model **does not understand chemistry** — it pattern-matches
  the literature corpus. Out-of-corpus systems are guesses.
